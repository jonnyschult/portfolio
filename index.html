Philosphy

Ux Ui

What makes a good user experience? Expectation vs experience often determines how the user feels about a website. As such, user experience is a dynamic process of expectation and interaction. Interactive cues can range from tacit, for example in a studio artist profile page, to explicit, in an e-commerce page. Whether they are tacit or explicit, they should always be apparent. My goal in developing websites is to meet and exceed user expectation by creating a website in-line with its purpose and aesthetic theme.

Code

There are many different ways to achieve the same results on a webpage. But quality programming goes beyond merely getting results once. Code should be transparent in it's purpose and logical structure, consistent, and deliver functionality effeciently. This is contingent upon an engineer's ability develop a cogent system which can functionally guide project development while maintaining a flexible approach which is responsive to unforseen design challenges. I aim to implement these ideals in my design strategies and coding.

Work

Business environments can be stressful. Often times this stress is compounded by poor working relationships. It is my philosophy that human productivity, and life in general, is best fostered in an enthusiastic, collaborative and open environment. When these things are done well, meaning and quality of life comes along with productivity. As a professional, I conduct my work in a way which brings meaning and productivity to myself and my colleagues.

Projects

RunnerSheets

RunnerSheets (working title) is an app in development designed to be used by athletic teams and running clubs. Essentially, it is a way for coaches and athletes, or club members to share activity information. This enables clubs to have peer to peer coaching and team coaches to track athlete fitness, particularly during the off season, and share feedback. Data sharing and communication form the bedrock of this apps development framework. This project relies on React, Node.js, PostgreSQL, Express.js, Sequelize, Bcrypt, JWT and Heroku.

BiblioQuest

Biblioquest is the product of a group project in a web development intensive course. My primary responsibilities were to design the back end for data persistence using Node.js, Express.js, Sequelize, Bcrypt, JWT and PostgreSQL, and migrating the data in the client from Google API to the server.
